## 2024-04-19 - Adding Keyboard Accessibility to Custom Interactive Elements
**Learning:** In projects heavily utilizing `div` elements for complex interactions (like the custom project accordion cards), keyboard navigation requires manual management. Without explicit `tabindex`, `role="button"`, `aria-expanded` and specific keydown event handlers ('Enter' and 'Space'), these interactive elements are completely inaccessible to keyboard users, breaking the usability of core features. Also, a global `:focus-visible` outline is critical to display the current keyboard focus without compromising the visual aesthetic for mouse users.
**Action:** When creating non-native interactive elements (like cards or accordions), always manually define the `role`, `tabindex`, dynamic `aria` attributes, and keyboard interactions (especially for 'Enter' and 'Space'). Furthermore, ensure a global `:focus-visible` style matching the design system is always defined.
