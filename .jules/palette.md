## 2024-03-24 - [Keyboard Accessibility for Interactive Divs]
**Learning:** In projects where non-native interactive elements (like `div` based accordions or project cards) are used, adding basic `click` listeners is not enough for keyboard users. To make these elements accessible, explicit `role="button"`, `tabindex="0"`, dynamic `aria-expanded` and `aria-controls` properties, and a `keydown` listener for 'Enter' and 'Space' keys must be added. Additionally, the global `*:focus-visible` must be set, especially in custom design systems where native focus styles might be hidden or clash with the design, so keyboard users can actually see where their focus is.
**Action:** Always check if interactive elements are native (`<button>`, `<a>`). If they are custom (`<div>`, `<span>`), explicitly add ARIA roles, tabindex, and keyboard event handlers. Furthermore, ensure `*:focus-visible` is defined globally in the CSS to make keyboard navigation visible.
