## 2025-05-18 - Keyboard Accessibility for Div-Based Accordions
**Learning:** Native `button` or `details`/`summary` elements provide built-in keyboard interaction, but when building custom interactive components like accordions or expanding cards using `div` elements, keyboard accessibility is completely missing by default. Setting `tabindex="0"` allows the element to receive focus, but it does not enable the `click` event when pressing Enter or Space like a native button would.
**Action:** Always ensure that non-native interactive elements (like custom `div`-based cards) are given `role="button"`, `tabindex="0"`, dynamic `aria-expanded` and `aria-controls` attributes. Most importantly, explicitly bind a `keydown` event listener targeting the 'Enter' and 'Space' keys, and ensure to call `e.preventDefault()` on 'Space' to prevent unintended page scrolling when interacting.
