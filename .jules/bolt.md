## 2024-05-24 - [Avoid redundant Math.sqrt in continuous gradients via preliminary threshold check]
**Learning:** This codebase uses `Math.sqrt` to calculate the continuous mathematical gradients required for fluid distortion effects (like proximity scaling). While `Math.sqrt` is expensive, it is necessary to produce smooth lerping values (like `influence`). Replacing it entirely with squared thresholds can disrupt these visual falloffs.
**Action:** Always derive squared thresholds (e.g. `CONFIG.influenceRadiusSq`) alongside original thresholds. In animation loops, use a preliminary squared distance check (`distSq < thresholdSq`) to gate the expensive `Math.sqrt` operation. This allows retaining the smooth visual gradients for elements within the influence radius while avoiding thousands of redundant square root operations per frame for elements outside the radius.
