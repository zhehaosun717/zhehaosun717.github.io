## 2024-06-25 - Conditional Execution of Math.sqrt for Continuous Gradients
**Learning:** When attempting to eliminate `Math.sqrt` calls from high-frequency animation loops (like `requestAnimationFrame`), replacing the distance check with squared variables (`distSq < radiusSq`) is effective. However, if the exact distance is still required later for calculating continuous mathematical gradients or smooth visual falloffs (such as linear interpolation or fluid distortion mapping), `Math.sqrt` cannot be completely removed.
**Action:** When refactoring distance checks in animation loops, wrap the `Math.sqrt` operation inside a conditional block (`if (distSq < thresholdSq)`) so it is only computed lazily when an element is within the active interaction radius, rather than globally per frame.
