## 2024-04-08 - Avoiding Math.sqrt and Layout Thrashing in Animation Loops
**Learning:** In performance-critical animation loops (like `requestAnimationFrame`), calculating `Math.sqrt` for distance continuous mathematical gradients and accessing layout properties like `window.innerHeight` on every frame can become bottlenecks. The former incurs redundant CPU cycles when objects are outside the effect radius, and the latter can trigger forced synchronous layouts (layout thrashing) when interleaved with DOM writes.
**Action:** Pre-calculate squared threshold constants (e.g. `influenceRadiusSq`) and use squared magnitude checks (`distSq < thresholdSq`) to conditionally bypass `Math.sqrt` when computing continuous visual falloffs. Furthermore, cache window dimensions and update them only within passive `resize` event listeners to eliminate frame-by-frame DOM reads.
