## 2024-04-16 - Pre-calculate squared distances for Three.js loops
**Learning:** In performance-critical animation loops containing hundreds of thousands of distance checks per frame (like connecting particles in `js/background.js`), calculating `distance * distance` repeatedly within the loop introduces unnecessary computational overhead. While modern JS engines optimize this, removing it from the tight `O(N^2)` inner loop provides measurable benefits and guarantees no jitter.
**Action:** Always pre-calculate squared threshold constants (e.g., `mouseDistanceSq = mouseDistance * mouseDistance`) outside the loop and compare them directly with the calculated squared distance (`distSq = dx*dx + dy*dy + dz*dz`). This eliminates redundant multiplication per frame and avoids costly `Math.sqrt` calls.
