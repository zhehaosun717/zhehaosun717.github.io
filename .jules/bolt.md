## 2026-04-18 - Caching Squared Thresholds in Three.js Loops
**Learning:** In highly iterated `requestAnimationFrame` loops in Three.js (e.g., iterating through thousands of particles per frame), repeatedly calculating static constant combinations like `mouseDistance * mouseDistance` inside the loop causes redundant computation overhead. Additionally, using `const` instead of `let` for variables holding window dimensions that are updated inside an event listener (like `resize`) throws assignment to constant variable errors, preventing the background from properly resizing and breaking the animation loop.
**Action:** When working on performance-critical animation loops or physics constraints, pre-calculate squared constants out of the loop and ensure mutability for dimension constraints attached to `resize` events.
