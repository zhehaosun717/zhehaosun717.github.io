## 2024-04-29 - Pre-calculating Distances in High-Frequency Loops\n**Learning:** In Three.js high-frequency animation loops, checking distances by multiplying base variables every frame (`mouseDistance * mouseDistance`) significantly increases redundant operations. Also, window bound variables like `windowHalfX` assigned outside the loop must be declared as `let` rather than `const` to allow recalculation inside `resize` event listeners, avoiding runtime `TypeError`s.\n**Action:** Always pre-calculate invariant squared threshold values outside of animation loops, and review mutability of any configuration variables shared between animation loops and resize listeners.
